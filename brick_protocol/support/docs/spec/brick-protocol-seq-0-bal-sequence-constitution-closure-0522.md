## 3. Active Sequence

The active sequence is:

```text
BrickWork boundary
-> ReceiptFact / AgentFact.returned inside that Brick work
-> optional BrickComparisonFact contract observation when comparison evidence exists
-> TransferFact / CarryFact / GateFact / MovementFact / TransitionFact between Brick boundaries
-> next BrickWork boundary
```

Short form:

```text
Brick -> Agent -> Brick -> Link -> Brick
```

This short form is explanatory only. The fact names above are the controlling
sequence for active docs and checker fixtures.

## 4. Brick And Agent Boundary

Brick owns the work contract:

```text
work_statement
comparison_rule
required_return_shape
source_facts
```

Agent receives and returns inside the Brick work boundary:

```text
ReceiptFact.received_work
AgentFact.received_work
AgentFact.returned
```

Link does not launch Agent, meet Agent, bind Agent, receive Agent returns, or
use Agent identity as an endpoint. The Brick / Agent relation may be projected
in a graph as Agent binding, but Agent binding is graph support vocabulary only,
not a new axis fact and not active constitutional sequence vocabulary.

## 5. BrickComparisonFact Boundary

`BrickComparisonFact` remains admitted and optional. It may record a contract
observation after returned facts exist and comparison evidence can be stated.

Allowed meaning:

```text
comparison_rule / required_return_shape observation
matched / missing / mismatched / unknown / blank observation shape
forbidden shortcut evidence
proof limits and not-proven context when carried by support evidence
```

Forbidden meaning:

```text
source truth
success judgment
failure judgment
approval
quality judgment
completion verdict
Movement authority
Link Gate replacement
```

This SEQ-0 slice freezes the current `BrickComparisonFact` shape. Any new field,
verdict literal, or required movement prerequisite needs a later Module
Admission Preflight.

## 6. Link Boundary

Link begins after public facts or work context exist at a Brick boundary and
continues only between Brick boundaries.

Link owns:

```text
TransferFact
CarryFact
GateFact
MovementFact
TransitionFact
```

Link may carry Agent-owned returned public fact refs, but it does not make the
Agent identity a source, target, endpoint, handoff target, or execution target.
Gate sufficiency facts may inform Movement, but Gate does not choose Movement.

SEQ-0 includes only narrow graph alignment:
