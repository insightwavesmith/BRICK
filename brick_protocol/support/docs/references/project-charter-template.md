# Project Charter Template

Use this neutral template before calling
`support.operator.project_creation.create_project`. Fill the six charter slots
in plain company language, confirm the text with the human owner, then stamp the
project vessel through the creation verb.

This template is support guidance only. It is not source truth, not success
judgment, not quality judgment, and not Movement authority.

## Project Identity

```text
project_id: <lowercase-slug-for-project-folder>
label: <human-readable project name>
declared_by: <human owner name>
```

## why_exists

Why does this project vessel exist?

```text
<one or more sentences naming the durable reason this project should have its
own project/<id>/ vessel>
```

## why_now

Why should this project be declared now?

```text
<one or more sentences naming the current trigger, urgency, or coordination
need>
```

## direction

Where is the project pointed?

```text
<one controlling direction sentence>
```

## done_means

What observable evidence means the project is making progress or can be closed?

```text
<facts, artifacts, or evidence conditions to observe; no success or quality
judgment>
```

## out_of_scope

What does this project explicitly not do?

```text
<boundaries, forbidden surfaces, deferred goals, or non-goals>
```

## managers

Who are the human owners?

```text
- <human owner name>
- <human owner name>
```

Managers are human owners only. Do not list Agents, provider names, model names,
credentials, setup tokens, or session identifiers.
