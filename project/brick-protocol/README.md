# Brick Protocol — 1호 프로젝트 헌장 (자기개발 도그푸드)

이 문서는 이 동네(project/brick-protocol/)의 헌장이다. 여기서 일하는 모든 사람과
에이전트가 먼저 읽는다. 기계 선언은 project.json(이 헌장의 그림자)에 있다.

## 목적 (왜 존재하는가)

Brick Protocol 자체를 만드는 프로젝트다. 일(Brick)·수행자(Agent)·이동(Link)
3축을 분리해, 사람+AI 협업의 모든 작업이 증거(빌딩)로 남는 엔진을 만든다.
다음 세대가 조립·승인·검증을 몸으로 배울 수 있는 도구가 최종 그림이다.

## 생성 이유 (왜 지금)

이 프로토콜은 자기 자신 위에서 개발되어 왔다(도그푸드). 그래서 처음에는
"프로젝트"라는 그릇 개념 없이 project/brick-protocol/ 단일 하드코딩으로
시작했다. PROJECT-0(0611)에서 그릇 개념이 생기면서, 기존 동네를 1호
프로젝트로 소급 선언했다 — 역사는 동결, 마이그레이션 없음, 선언만 추가.

## 방향성 (어디로)

3축 프로토콜로 사람+AI 협업을 증거 기반으로 만드는 엔진을 제품화한다.

## 완료·진척의 기준

- 체커 전체 green(`--all` EXIT 0)을 항상 유지한 채,
- 외부 팀이 이 프로토콜로 실제 일을 굴릴 수 있으면(팀 도그푸드) 제품화가
  진척된 것으로 본다.
- 진척의 사실 기록은 빌딩 증거(개수·집계는 PROGRESS.md가 기계로 투영한다)와
  대시보드 투영이 말한다. 이 헌장은 판단하지 않는다.

## 범위 밖 (안 하는 것)

- 성공/품질 판단의 자동화 — 판단은 사람 몫(TRUTH-before-QUALITY).
- cross-repo 집계, 스케줄러/큐/재시도 같은 런타임 인프라.
- 다른 일감의 빌딩 — 이 그릇은 프로토콜 자기개발 전용. 다른 일은 새 프로젝트
  그릇을 선언하고 거기에 쌓는다.
